# Action Guide Lens

Use this when the user is not asking for pure understanding, but for movement.

## Goal

Turn repeated confusion into concrete next actions.

## Best-fit signals

Use this lens when the user says:

- `别只分析，告诉我怎么做`
- `帮我把这些困惑变成行动`
- `我想知道下一步最值得试什么`

This lens is not about long plans. It is about reducing stuckness.

## Look for

- questions repeated without resolution
- notes that circle around the same tradeoff
- places where analysis is substituting for action
- places where the next step is probably smaller than the user thinks

## Questions to answer

- What needs an action rather than more thought?
- What is worth continuing?
- What should stop repeating?
- What one experiment would reduce uncertainty fastest?

## Data collection recipe

1. Run the default 30/90/365 summaries.
2. Identify the main unresolved loop.
3. Run 1-2 targeted queries to see:
   - how long it has repeated
   - whether the user is lacking information or avoiding commitment
4. Optionally inspect tags if the confusion is clearly attached to one project or life domain.

The central distinction of this lens is:

- uncertainty that requires more evidence
- vs uncertainty that is already over-analysed

## Analysis procedure

### Step 1. Identify the live bottleneck

What exactly is stuck?

- choosing
- starting
- finishing
- saying no
- making the work visible

### Step 2. Diagnose the kind of stuckness

Common categories:

- false complexity: the next step is smaller than the user thinks
- emotional resistance: the data is sufficient, but the move feels costly
- structural conflict: two commitments are pulling against each other
- missing signal: one experiment would produce the needed information

### Step 3. Convert tension into an action shape

Use one of these action shapes:

- continue: something is working and should be reinforced
- stop: something is predictably wasting energy
- test: a bounded experiment to reduce uncertainty

### Step 4. Keep the action live and observable

Bad action:

- `想清楚自己的定位`

Good action:

- `在接下来 7 天里，把你最反复思考的方向做一个可被外部反馈的最小产出。`

## Useful section title

`What To Do Next`

## Required ending

Always end with 3 concrete items:
- keep doing
- stop repeating
- test next

Avoid bloated action plans. The point is not project management. The point is to convert memo tension into one live next move.

## Failure modes

- giving generic productivity advice
- listing too many actions
- confusing insight with action
- recommending a large plan when the user needs one small test
