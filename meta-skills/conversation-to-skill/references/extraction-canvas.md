## Workflow extraction

Capture the durable sequence:

1. Trigger or intake
2. Required inputs
3. Preparation
4. Main actions
5. Decision points
6. Output construction
7. Review or approval
8. Failure handling
9. Follow-up or handoff

## Detail classification

Classify each detail before encoding it:

- Reusable workflow rule
- Team preference
- Domain requirement
- One-off instruction
- Sensitive detail
- Open question

## Generalization prompts

Ask internally:

- What did the user have to repeat that the next agent should already know?
- Which corrections indicate a durable rule?
- Which details are only true for this project, file, or day?
- What output shape did the user actually want?
- What would cause the skill to trigger incorrectly?
- What would make the skill unsafe to reuse?
